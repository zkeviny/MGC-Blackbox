# 数据分析知识沉淀（Knowledge Management）

本文档用于指导数据分析师在 **MGC Blackbox** 中安全管理分析框架、提示词模板、业务规则、方法论等知识资产，实现本地加密存储与零暴露访问。

所有敏感操作必须由用户明确授权后才能进行。

---

# 📌 一、知识管理概述

MGC Blackbox 提供知识内容的本地加密存储与零暴露访问能力：

- 知识内容永不暴露给 AI  
- 知识仅在用户本地机器解密  
- AI 可在用户授权后读取知识，但不可见明文  
- 所有访问必须由用户授权  

知识沉淀是构建可复用分析体系的核心能力。

---

# 📌 二、可存储的知识类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 分析框架 | 标准化分析结构 | “月度销售分析框架” |
| 提示词模板 | 可复用的提示词 | “数据质量检查提示词” |
| 方法论 | 分析方法步骤 | “队列分析方法论” |
| 业务规则 | 领域规则 | “客户分层规则” |
| 最佳实践 | 经验总结 | “数据验证最佳实践” |

所有知识内容必须由用户提供，本套件不生成知识内容。

---

# 📌 三、如何安全存入知识内容

知识存储推荐使用 **MGC WebUI** 或 **mgc_save** 工具。

---

## ✅ 方法 1：通过 WebUI 存储（推荐）

1. 启动 MGC：  
   ```
   mgc
   ```
2. 打开 WebUI：  
   ```
   http://127.0.0.1:57218
   ```
3. 进入 **Save 页面**  
4. 填写字段：

```
info_type: "prompt" 或 "knowledge"
info_owner: "framework_monthly_sales"   # 自定义名称
content: <你的知识内容>
```

---

## ✅ 方法 2：通过 mgc_save 存储

```python
mgc_save(
    info_type="prompt",
    info_owner="framework_monthly_sales",
    content="""
# 月度销售分析框架
## 步骤 1：数据验证
- 检查数据完整性
- 验证日期范围

## 步骤 2：关键指标
- 总收入
- 增长率
- 客户数
"""
)
```

# 📌 四、如何安全读取知识内容（零暴露）

AI 可在用户授权后读取知识，但不可见明文内容。

---

## 🔐 标准读取方式（需用户授权）

```python
knowledge = mgc_get(
    info_type="prompt",
    info_owner="framework_monthly_sales"
)
```

### 🔒 安全特性

- AI 不可见知识明文  
- 所有访问在本地完成  
- 所有访问必须由用户授权  

---

# 📌 五、如何在分析中使用知识内容

知识内容可用于：

- 构建分析报告结构  
- 指导分析步骤  
- 生成提示词  
- 复用业务规则  
- 复用方法论  

示例（AI 在用户授权后使用知识）：

```
用户：“使用月度销售分析框架帮我分析这个数据。”

→ AI 请求授权读取知识
→ 用户授权
→ AI 使用 knowledge 内容构建分析结构
```

---

### 🔒 安全特性

- 接收方可在本地读取知识  
- 知识内容永不暴露  

---

# 📌 七、授权模板（AI 必须询问）

当 AI 需要读取知识时，必须询问用户：

```
知识名称：framework_monthly_sales
操作：读取知识内容
是否需要授权：是

您是否授权读取此知识？回复“是”继续，回复“否”取消。
```

AI 在未获得用户授权前不得读取任何知识内容。

---

# 📌 八、最佳实践

### ✔ 1. 使用描述性知识名称  
例如：

- `framework_monthly_sales`  
- `prompt_data_quality_check`  
- `rule_customer_segmentation`

### ✔ 2. 知识内容应结构化  
例如：

- Markdown  
- JSON  
- 分步骤结构  

### ✔ 3. 不在 AI 对话中粘贴知识明文  
AI 无法保护敏感内容。

### ✔ 4. 所有读取必须授权  
AI 不得自动读取知识。

# 📌 九、常见问题

| 问题 | 解决方案 |
|------|---------|
| 无法读取知识 | 检查 info_owner 是否一致 |
| 知识为空 | 检查 WebUI 是否正确存储 |
| 密封失败 | 检查目标节点是否安装 MGC 1.4.6+ |
