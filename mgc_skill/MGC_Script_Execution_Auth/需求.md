目标：以脚本0暴露加密授权为核心，基于MGC的脚本加密存储/密封功能和密封脚本灰盒运行作为底层依托，创建0暴露加密授权skill，贴合0暴露系列命名；
目标store：skillhub、clawhub、modelscope
skill文件结构：
- readme
- SKILL.md
- manifest.json
执行文件：为了方便过审和降低审核通过难度，最好不用mcp server和example.py；
SKILL要求：
- 减少代码引用，但要明确需要安装mgc-blackbox，最好写明pip install mgc-blackbox；
- 技能命名：Cross‑Device Encrypted Script Authorization（Zero‑Exposure） / 脚本跨设备加密授权(零暴露)
- 技能ID：cross_node_script_auth
- 重点介绍使用场景和技能价值
SKILL描述：
 1、应用场景：将本地MGC能运行的脚本通过RSA密封，交给对应节点（需提供对应节点的node_pub），对应节点存入MGC后即可运行，若依赖配置和脚本，则需约定好依赖的info_type和info_owner一致性，否则会因为依赖原因无法正确运行；
 2、使用流程：申请节点安装MGC后，通过mgc_get的mcp服务，获取node_pub（若没有会自动生成），将node_pub给到脚本归属节点（脚本归属人），脚本归属节点通过mgc_seal并传入申请节点的node_pub，即可获得密封后的完整密文信息，将信息原封不动的给到申请节点，申请节点存入MGC，即可运行，整个过程中脚本在传输和被授权节点的MGC中始终都是密文状态，完全零暴露；
 3、注意事项：
  - 密封脚本在进程中沙盒运行，无法规避恶意root权限，需用户自己识别被授权方是否可信；
  - 若遇到无法运行的情况，请先确认是否依赖的配置信息不正确，或MGC版本低于1.4.6；
  - 密封信息需按技能文档要求存入（区分AI或webui存入的两种情况）；
